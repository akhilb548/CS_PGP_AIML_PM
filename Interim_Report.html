Executive Summary

Project Overview and Key Findings

Purpose of This Report

This interim report documents the first phase of the Engine Predictive Maintenance project — a machine learning initiative designed to predict engine failures before they occur. The report covers data registration, exploratory analysis, data preparation, and the initial model-building phase. It is intended for senior decision-makers evaluating the technical progress and business potential of this initiative.

Business Problem

Unplanned engine failures cost fleet operators and manufacturers millions of dollars annually through emergency repairs, vehicle downtime, and safety incidents. Traditional time-based maintenance schedules are inefficient — they either service engines too early (wasting resources) or too late (risking failure). This project builds a data-driven predictive maintenance system that uses real-time engine sensor readings to flag engines at risk of failure before a breakdown occurs.

Key Business Findings

  • Engine RPM is the strongest predictor of failure — faulty engines run at significantly lower RPMs (736 vs. 885 RPM on average), indicating mechanical stress or degradation.
  • Coolant Temperature anomalies are critical warning signs — extreme outliers reaching 195°C (vs. a normal range of ~62–90°C) signal imminent overheating failures.
  • The XGBoost model achieves 94.2% accuracy with a 95.3% recall rate, meaning the system correctly identifies 95 out of every 100 faulty engines — minimizing costly missed detections.
  • All models and datasets are version-controlled on Hugging Face Hub, enabling reproducibility and future deployment.

Business Impact Potential

Deploying this predictive model in a fleet management or manufacturing context can reduce unplanned downtime by an estimated 30–50%, lower emergency repair costs, and extend engine lifespan through timely, targeted maintenance interventions.

Data Registration

Data registration is the foundational step that ensures all project assets — raw data, processed datasets, and trained models — are systematically organized, versioned, and accessible to all project stakeholders. This section describes how the project's file structure was established and how the dataset was registered on a centralized data platform.

1.1 Project Folder Structure

A standardized folder hierarchy was created programmatically using Python's pathlib library, ensuring consistent organization across all environments (local development, cloud, CI/CD pipelines). The structure separates raw data, processed outputs, trained models, and experiment logs into dedicated directories.

1.2 Dataset Registration on Hugging Face Hub

The raw engine sensor dataset was registered on the Hugging Face Dataset Hub — an industry-standard platform for versioned dataset management. This provides several business benefits:

  • Version Control: Every dataset update is tracked with a commit history, enabling rollback to any prior state.
  • Accessibility: The dataset can be loaded programmatically from any environment using a single API call, eliminating manual file transfers.
  • Reproducibility: All team members and automated pipelines reference the same canonical dataset, preventing data drift between environments.
  • Governance: Access controls on Hugging Face ensure only authorized users can modify the registered dataset.

Exploratory Data Analysis (EDA)

2.1 Data Collection and Business Background

The dataset was sourced from engine sensor monitoring systems deployed across a fleet of vehicles and industrial machinery. Each record represents a snapshot of an engine's operating state at a given point in time, captured by six onboard sensors. The target variable — Engine Condition — is a binary label indicating whether the engine is operating normally (0) or is in a faulty/at-risk state (1).

Table 2.1 — Feature Descriptions and Business Relevance
Feature Description Unit Business Relevance
Engine_RPM Engine revolutions per minute RPM Primary indicator of engine load and mechanical health
Lub_Oil_Pressure Lubricating oil pressure bar / kPa Low pressure signals lubrication failure → bearing damage
Fuel_Pressure Fuel supply pressure bar / kPa Abnormal pressure indicates fuel system faults
Coolant_Pressure Engine coolant pressure bar / kPa Pressure drops signal coolant leaks or pump failure
Lub_Oil_Temperature Lubricating oil temperature °C Elevated temperature degrades oil viscosity and protection
Coolant_Temperature Engine coolant temperature °C Critical overheating indicator — extreme values signal imminent failure
Engine_Condition Engine health label (Target) 0 = Normal, 1 = Faulty Prediction target for the maintenance alert system
